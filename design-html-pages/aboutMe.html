How I Like to Code

I prefer to code to a standard first, make one very high quality route for example, then use that as an example for the rest of the project.

For example, when refactoring Topside Tipoff's backend into Nest.js I first took a simple group of routes and did that one to the best of my abilities, asking as many questions about how it should be done as possible before moving onto another set of routes and using the first one as an example.
